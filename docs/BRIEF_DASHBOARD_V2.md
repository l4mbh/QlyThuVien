## 🎯 1. MỤC TIÊU (REDEFINED)
Dashboard không chỉ là nơi xem số liệu, mà là **Trung tâm điều hành**. Mục tiêu cao nhất là giúp người dùng (Admin/Staff) biết ngay lập tức các vấn đề cần xử lý trong ngày.

## 👥 2. PHÂN QUYỀN (ROLE-BASED)
- **Admin:** Thấy toàn bộ, bao gồm số liệu tài chính (Fines).
- **Staff:** Tập trung vào tác nghiệp (Overdue, Low stock), ẩn các số liệu doanh thu nhạy cảm.

## 🚀 3. CẤU TRÚC GIAO DIỆN (THE LAYOUT)

### 🧩 3.1. TOP: Summary Cards (Signals)
- **Books**: Total quantity.
- **Borrows**: Active borrowing count.
- **Overdue**: Red alert if > 0.
- **Fines**: Total amount (Admin only).

### 🧩 3.2. MIDDLE: Action Panels (The Real Work)
- **Overdue Table**: Chi tiết reader nợ sách, sắp xếp theo số ngày quá hạn giảm dần.
- **Low Stock Table**: Sách có `availableQuantity < 3`.

### 🧩 3.3. BOTTOM: Charts (Insights)
- **Trend**: Line chart mượn trả 7 ngày.
- **Top Books**: Danh sách top 5.

## 🛠️ 4. DATA FLOW & APIs
Backend sẽ cung cấp các API chuyên biệt:
- `GET /reports/summary` (Refactored)
- `GET /reports/overdue` (Detailed for dash)
- `GET /reports/low-stock` (New)
- `GET /reports/borrow-trend`
- `GET /reports/top-books`

## ✨ 5. UX/UI STANDARDS
- **Loading:** Skeleton screens.
- **Empty State:** "All clear" messages.
- **Error:** Retry capability.
- **Layout:** Responsive (4 columns -> 2 columns -> 1 column).
