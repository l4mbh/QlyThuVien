## 1. Authentication Flow (Phone Identity)
Thay đổi cơ chế định danh từ `localStorage.reader_phone` thuần túy sang JWT.

### Sequence:
1.  **Reader App**: User nhập SĐT -> `POST /api/v1/auth/reader-login { phone: string }`.
2.  **Backend**:
    *   Normalize SĐT (+84...).
    *   `findOrCreateUser` (Role: READER, isGuest: true, hasActivity: false).
    *   Generate JWT Token.
    *   Trả về `{ user, token }`.
3.  **Reader App**: Lưu `token` vào `localStorage`. Các request sau đính kèm Header `Authorization: Bearer <token>`.

## 2. Dynamic Category Bar
*   **API**: `GET /api/v1/categories`.
*   **Frontend**: Sử dụng `useQuery` để fetch và cache danh sách. Xóa bỏ mảng `categories` hardcoded trong `CategoryBar.tsx`.

---

## 3. Smart Borrow Button
Nút bấm tại `BookDetailModal.tsx` sẽ hoạt động theo State:
*   `!isLoggedIn`: Text "Sign in to Borrow" -> Redirect login.
*   `isLoggedIn && availableQuantity > 0`: Text "Borrow this Book" -> Gọi API borrow.
*   `isLoggedIn && availableQuantity <= 0`: Text "Join Queue" -> Gọi API reserve.

---

## 4. Database Changes
*   **User Model**:
    *   `isGuest`: Boolean (default true).
    *   `hasActivity`: Boolean (default false).
    *   `phoneNormalized`: String (unique).

## 5. API Contract
*   `POST /auth/reader-login`: Public.
*   `GET /categories`: Public.
*   `POST /borrow`: Protected (READER/STAFF/ADMIN).
